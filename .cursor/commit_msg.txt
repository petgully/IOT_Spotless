Phase 3: operator admin web UI at /admin (config editor + system health)

Booth managers can now change shampoo/water/dryer timings, geyser schedule,
and roof-light window from a phone or laptop instead of editing JSON on the
Pi. Mounted into the existing kiosk Flask app, gated behind HTTP Basic Auth
using SPOTLESS_ADMIN_PASSWORD from .env.

New: raspberry_pi/admin/
- operator_auth.py - constant-time password check; default-password warning
- operator_validators.py - bounds + HH:MM checks (no bad values reach disk)
- operator_routes.py - Flask Blueprint with /admin/, /admin/settings,
  per-section form POSTs, JSON API mirror, and unauthenticated /admin/healthz
- operator_templates/ - dark-themed dashboard + settings UI (mobile-friendly)
- operator_static/ - self-contained CSS, no CDNs

Hot-reload plumbing so admin saves apply without restarting the kiosk:
- config_manager.py: atomic save (temp + os.replace), reload() helper, and
  _save_to_local now raises on disk failure so callers can surface real
  errors instead of silently flashing "saved".
- kiosk/web_server.py: _profile_overrides() force-reloads from disk on
  every QR scan so the next session uses the operator's new timings.
- geyser_controller.py + roof_light_controller.py: apply_config() updates
  scheduler attributes in place; in-progress cycles finish on old values,
  next cycle uses new ones.

Operator UX:
- Save errors (full SD card, permission issue) now show as a red flash
  message with the actual exception text - no more silently flashing OK.
- Default-password banner warns operators they're still on spotless-admin.
- Form submissions use PRG (303 redirect) with anchor jumps so the page
  scrolls back to the section the operator just saved.

Existing /admin relay-test dashboard on :8080 (admin_server.py) is left
untouched - different blueprint, different folders, zero collision.

Docs:
- DEPLOYMENT.md: dedicated /admin section, day-to-day ops table updated to
  point at the web UI instead of JSON editing, Phase 3 marked Done.
- bootstrap.sh print_summary now prints the Admin URL after first boot.
- .env.example explains SPOTLESS_ADMIN_PASSWORD purpose and rotation.
