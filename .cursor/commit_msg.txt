Phase 2a: Windows ESP32 flash wrapper (tools/flash_node.ps1)

One-command ESP32-S3 flashing for non-technical operators on Windows.
Replaces the manual "edit config.h, find PlatformIO PATH, detect COM port,
cd into node dir, run pio" sequence with:

    .\tools\flash_node.ps1 1                # flash node 1
    .\tools\flash_node.ps1 all              # flash all 3 (prompts to swap USB)
    .\tools\flash_node.ps1 all -WifiSsid X -WifiPassword Y -MqttBroker Z   # site setup
    .\tools\flash_node.ps1 -Check           # toolchain sanity check

Wrapper features:
- Auto-locates pio.exe regardless of where Python is installed
  (PATH -> ~/.platformio/penv -> %APPDATA%/Python/Python*/Scripts -> ...).
  No more hardcoded Python313 path in the agent skill.
- Auto-detects ESP32 COM port; filters out Bluetooth virtual ports.
  Pre-validates user-supplied -Port against actual present ports.
- Reads + displays config.h (NODE_ID / WIFI_SSID / MQTT_BROKER, masked
  password) before flashing so the operator confirms intent.
- Optional in-place patching of WiFi/MQTT settings via flags
  (uses [regex]::Replace with proper capture-group escaping).
- 'all' mode prints the physical role of each board (Container 1 / 2 /
  Bath line) in the swap prompt so identical-looking PCBs aren't
  cross-flashed.
- 'all' mode always re-detects COM port between boards (the port almost
  always changes); -Port is intentionally ignored there.
- -Check returns exit 1 if no ESP32 is detected (was incorrectly 0).
- ASCII-only output (works in PS 5.1 default ANSI encoding without BOM).

Companion docs:
